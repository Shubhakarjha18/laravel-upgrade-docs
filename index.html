Upgrading Laravel Version from any version to latest version

Upgrade Steps

1

Backup

Backup the full project including:

  • Source code
  • Database
  • composer.json configuration file
2

Update composer.json File

Replace the current composer.json file with the Laravel 11.x compatible version.

View Laravel GitHub Repository
3

PHP Version Change

Upgrade PHP to version 8.x compatible with Laravel 11.x.

4

Update Dependencies

Run the following command to install/update dependencies:

composer update
5

Configuration Changes

  1. Remove cors.php
    If you are using a custom cors.php, remove it because Laravel 11.x uses a new configuration approach.
  2. Regenerate CORS Configuration
    Run:
    php artisan config:publish cors
  3. Update Kernel.php
    Remove:
    \Fruitcake\Cors\HandleCors::class
  4. Update TrustProxies.php
    Replace:
    use Fideloper\Proxy\TrustProxies as Middleware;
    With:
    use Illuminate\Http\Middleware\TrustProxies as Middleware;
    Update protected headers:
    protected $headers = Request::HEADER_X_FORWARDED_FOR | Request::HEADER_X_FORWARDED_HOST | Request::HEADER_X_FORWARDED_PORT | Request::HEADER_X_FORWARDED_PROTO | Request::HEADER_X_FORWARDED_AWS_ELB;
  5. Remove Swift Mailer
    Laravel 11 replaces SwiftMailer with Symfony Mailer. Remove SwiftMailer references from notification files and test all mailing functionalities.
6

Run Project & Test Functionalities

  1. Update media table by adding generated_conversion field through migration.
  2. Install Auth UI package after upgrade.
  3. Test all project functionalities and fix issues if found.
  4. Remove Spatie permission class from config/app.php.
  5. Update role middleware references inside Kernel.php.
